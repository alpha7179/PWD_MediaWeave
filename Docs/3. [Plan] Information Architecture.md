# Information Architecture

## 개요

Media Weave 서비스의 정보 구조 및 네비게이션 체계 정의. 본 문서는 모든 페이지의 URL 구조, 권한 체계, 네비게이션 흐름을 명시하여 프론트엔드 개발자가 즉시 라우팅 구현 가능한 수준의 상세 명세 제공.

## 범례

- `[Public]`: 비로그인 사용자 및 외부 방문자 접근 가능
- `[Student]`: 인증된 학생 회원만 접근 가능
- `[Professor]`: 교수 권한 보유자만 접근 가능
- `[Admin]`: 시스템 관리자 및 학과 조교 전용 권한
- `(Action)`: 클릭, 드래그 등 사용자의 직접적인 행동
- `(System)`: 데이터 로딩, 계산, 필터링 등 시스템의 백그라운드 처리
- `(OCR System)`: 이미지 인식 및 텍스트 추출 자동화 프로세스
- `(Feedback)`: 사용자 행동에 대한 시각/청각적 피드백 (Toast, Modal, Animation)

## URL 구조 정의

### 인증 관련

| URL | 메서드 | 파라미터 | 쿼리스트링 | 권한 | 설명 |
|-----|--------|----------|-----------|------|------|
| `/auth/login` | GET | - | `redirect?: string` | [Public] | 로그인 페이지 |
| `/auth/register` | GET | - | `type?: student\|professor` | [Public] | 회원가입 페이지 |
| `/auth/logout` | POST | - | - | [Student, Professor, Admin] | 로그아웃 처리 |
| `/auth/reset-password` | GET | - | `token?: string` | [Public] | 비밀번호 재설정 |

### 메인 및 대시보드

| URL | 메서드 | 파라미터 | 쿼리스트링 | 권한 | 설명 |
|-----|--------|----------|-----------|------|------|
| `/` | GET | - | - | [Public] | 랜딩 페이지 (비로그인) |
| `/dashboard` | GET | - | - | [Student, Professor] | 사용자 대시보드 (로그인 후) |

### 커리큘럼 관련

| URL | 메서드 | 파라미터 | 쿼리스트링 | 권한 | 설명 |
|-----|--------|----------|-----------|------|------|
| `/curriculum/roadmap` | GET | - | `track?: DE\|CD\|GI\|MD\|VC` | [Public] | 트랙 로드맵 뷰어 |
| `/curriculum/roadmap/course/:courseId` | GET | `courseId: string` | - | [Public] | 과목 상세 모달 |
| `/curriculum/planner` | GET | - | - | [Student] | 나만의 커리큘럼 플래너 |

## 리다이렉션 규칙

### 인증 기반 리다이렉션

| 조건 | 현재 URL | 리다이렉션 대상 | 우선순위 |
|------|----------|----------------|----------|
| 비로그인 상태에서 인증 필요 페이지 접근 | `/curriculum/planner` | `/auth/login?redirect=/curriculum/planner` | 1 |
| 비로그인 상태에서 인증 필요 페이지 접근 | `/schedule` | `/auth/login?redirect=/schedule` | 1 |
| 로그인 상태에서 인증 페이지 접근 | `/auth/login` | `/dashboard` | 2 |
| 로그인 상태에서 랜딩 페이지 접근 | `/` | `/dashboard` | 2 |
| 학생이 교수 전용 페이지 접근 | `/admin/*` | `/dashboard` (403 에러) | 3 |
| 권한 없는 프로필 편집 시도 | `/profile/:userId/edit` | `/profile/:userId` (403 에러) | 3 |

## 1. Global Navigation Bar (GNB)

> 서비스의 모든 페이지에서 사용자가 길을 잃지 않도록 돕는 나침반 역할을 하며, 모바일 환경에서는 햄버거 메뉴(Hamburger Menu)로 반응형 처리됩니다.
> 
- **Logo (Home)**
    - (Action: 클릭 시 메인 대시보드(로그인 후) 또는 랜딩 페이지(로그인 전)로 이동하여 탐색 상태 초기화)
- **미디어 커리큘럼 (Curriculum Hub)**
    - (Hover Action: 하위 메뉴 드롭다운 노출 - 마우스 오버 시 0.2초 딜레이 후 부드럽게 펼쳐짐)
    - **Sub Menu:** 트랙 로드맵, 나만의 커리큘럼, 공유된 커리큘럼
- **스마트 시간표 (Smart Schedule) `[User]`**
    - (Hover Action: 하위 메뉴 드롭다운 노출)
    - **Sub Menu:** 내 시간표 보기, 외부 캘린더 연동 설정
    - (Badge: '수업 10분 전' 등 긴급 알림 뱃지 표시)
- **커뮤니티 (Community)**
    - (Hover Action: 하위 메뉴 드롭다운 노출)
    - **Sub Menu:** 프로젝트 갤러리(아카이브), 팀 빌딩(구인구직)
- **우측 유틸리티 영역**
    - **통합 검색 (Global Search):** 프로젝트, 과목명, 사용자, 태그를 아우르는 전역 검색
    - **알림 (Notification):** 팀 초대, 댓글, 공지사항 알림 (읽지 않은 알림 개수 표시)
    - **프로필 (Profile):** 사용자 썸네일 표시, 클릭 시 '마이페이지' 및 '로그아웃' 메뉴 노출

## 2. Page Structure (상세 구조)

### 1. Home (메인)

- **Depth 1: Landing Page (로그인 전) `[Public]`**
    
    > 신규 방문자에게 서비스의 핵심 가치(Value Proposition)를 설득하고 가입을 유도하는 쇼케이스 영역입니다.
    > 
    - **Depth 2: 히어로 섹션 (Hero Section)**
        - (Visual: '복잡한 커리큘럼'이 '정돈된 로드맵'으로 변하는 애니메이션)
        - (Action: 메인 CTA 버튼 "내 커리큘럼 시작하기" 클릭 → 회원가입 모달 팝업)
    - **Depth 2: OCR 기능 하이라이트 (Feature Demo)**
        - (Action: "성적표/시간표 1초 만에 등록하기" 시연 영상 재생 - 수동 입력 대비 시간 절약 강조)
        - (Action: **OCR 맛보기 체험** - 사용자가 샘플 이미지를 드래그 앤 드롭하면 즉시 텍스트가 파싱되는 과정을 시각화하여 신뢰도 확보)
    - **Depth 2: 소셜 증명 (Social Proof)**
        - (System: "현재 000명의 디미과 학생들이 사용 중입니다" 실시간 카운터 표시)
        - (System: 명예의 전당에 등록된 우수 프로젝트 썸네일 롤링 배너)
- **Depth 1: User Dashboard (로그인 후) `[User]`**
    
    > 사용자의 현재 학업 상태를 요약하고, 즉시 필요한 행동(Action Item)을 제안하는 개인화 대시보드입니다.
    > 
    - **Depth 2: 학업 현황 요약 (Academic Status)**
        - (OCR System: **성적표 파싱 데이터 기반** 누적 평점(GPA), 전공/교양 이수 학점 도넛 차트 표시)
        - (System: 설정한 '주력 트랙(예: GI)'의 이수율 계산 - "졸업까지 전공선택 3과목 남았습니다.")
        - (Action: '상세 분석 보기' 클릭 시 마이페이지의 성적 관리 탭으로 이동)
    - **Depth 2: 오늘의 일정 (Daily Briefing)**
        - (OCR System: **시간표 파싱 데이터 기반** 현재 시간 기준 가장 가까운 수업 정보 표시)
        - (Display: 과목명, 강의실 위치(호수), 교수님 성함, 수업 시작까지 남은 시간 타이머)

### 2. Media Curriculum (미디어 커리큘럼)

- **Depth 1: 트랙 로드맵 (Track Roadmap) `[Public]`**

        - **Depth 3: 과목 상세 모달 (Course Detail Modal)**
            - (System: **학과 요람 OCR 데이터 연동** - 과목 개요, 학점, 설계/이론/실습 배점, 개설 학기 정보 자동 표시)

- **Depth 1: 나만의 커리큘럼 (Planner) `[User]`**
    
    > 사용자가 자신의 8학기 수강 계획을 시뮬레이션하고 검증하는 도구입니다.
    > 
    - **Depth 2: 플래너 편집기 (Editor Interface)**
        - **Depth 3: 성적표 불러오기 (Import Transcript with OCR)**
            - (Action: '성적표 업로드' 버튼 클릭 → AIMS 포털 성적표 캡처 이미지/PDF 선택)
            - (OCR System: 이미지 전처리 → 텍스트 추출 → **[년도/학기/과목명/이수구분/학점/성적]** 구조화 파싱)
            - (Interaction: **검증 인터페이스(Verification UI)** - 원본 이미지와 추출 텍스트를 나란히 보여주며, 인식 신뢰도가 낮은 항목은 사용자에게 확인 요청)

            - 개설 학기 불일치 경고: 1학기 개설 과목을 2학기에 배치 시 알림.

        - (Action: 선배들의 커리큘럼을 학번별, 트랙별, 취업 분야별로 필터링하여 조회)
