# 1. 프로젝트 개요 (Project Overview)

## 1.1. 목표 (Goal)

본 프로젝트의 일차적 목적은 아주대학교 디지털미디어학과 재학생의 5개 자율 트랙(DE, CD, GI, MD, VC) 중 적성과 진로에 부합하는 커리큘럼 주도적 설계를 위한 명확한 정보(로드맵) 및 구체적 학업 성과(프로젝트) 제공 및 공유.

궁극적 목표는 학과 내 프로젝트의 체계적 아카이빙 및 팀 빌딩(Team Building) 프로세스 공식화를 통한 파편화된 학술 자산 집대성 및 협업 품질 향상. 이를 통한 재학생 포트폴리오 질적 향상, 협업 프로젝트 성공률 제고, 학과 교육 목표와 산업 현장 요구 간극 축소를 위한 핵심 플랫폼 구축.

### 정량적 목표 (Quantitative Goals)

**1학기 목표 (MVP 출시 후 4개월)**
- 재학생 가입률: 30% (약 150명 / 전체 재학생 500명 기준)
- 커리큘럼 생성 건수: 100건 이상
- 프로젝트 등록 건수: 50건 이상
- 팀 빌딩 게시물 작성 건수: 30건 이상
- 주간 활성 사용자(WAU): 80명 이상

**2학기 목표 (출시 후 8개월)**
- 재학생 가입률: 60% (약 300명)
- 커리큘럼 생성 건수: 250건 이상
- 프로젝트 등록 건수: 150건 이상
- 팀 빌딩 게시물 작성 건수: 80건 이상
- 주간 활성 사용자(WAU): 180명 이상

**핵심 성과 지표 (KPI)**
- 사용자 참여율: 월 1회 이상 로그인 사용자 비율 ≥ 70%
- 커리큘럼 완성도: 4학기 이상 계획 작성 비율 ≥ 60%
- 프로젝트 품질: 평균 조회수 ≥ 50회, 평균 '좋아요' ≥ 5개
- 팀 빌딩 성공률: 모집 완료 상태 전환 비율 ≥ 40%
- 사용자 만족도: NPS(Net Promoter Score) ≥ 50

## 1.2. 문제 정의 (Problem Statement)

본 서비스의 해결 대상은 디지털미디어학과 학생이 공통적으로 직면하는 네 가지 핵심 문제.

1. **높은 자율성에 따른 정보 비대칭** 
    - 디지털미디어학과의 전공필수 과목(5개) 수는 상대적으로 적으며, 다수의 전공선택 과목 운영으로 학생의 자율적 수강 설계 요구. 그러나 5개 권장 트랙과 과목 간 연관성에 대한 명시적 가이드라인 부재.
    - **결과:** 정보 비대칭으로 인한 학생의 진로 설계 심리적 불안감 가중 및 최적 학습 경로 탐색 시 과도한 시행착오 비용 발생.
2. **교과목 정보의 불투명성**
    - 현행 학과요람 PDF의 과목 텍스트 기반 개요만 제공. 실제 수강 난이도, 트랙정보, 활용 기술 스택, 최종 산출물 수준 파악의 한계.
    - **결과:** 학생의 기대 및 진로 목표와 상이한 과목 선택 리스크 노출 및 효율적 기술 스택 구축 저해.
3. **학습 성과의 파편화**
    - 과목 이수를 통한 우수 프로젝트 결과물의 개인 GitHub, Notion 등 외부 플랫폼 파편화 및 체계적 아카이빙 부재.
    - **결과:** 학과 공동 지적 자산 유실 및 후배 재학생의 선배 성과 참고를 통한 학습 질 향상 기회 상실.
4. **비효율적인 팀 구성 프로세스**
    - 미디어프로젝트 등 협업 기반 프로젝트 교과목에서 특정 기술 스택 보유 팀원 모집을 위한 '에브리타임' 등 비공식 익명 커뮤니티 의존.
    - **결과:** 팀 구성 과정의 불필요한 시간 소모 및 마찰 발생, 역량 기반 객관적 팀 매칭 부재로 인한 프로젝트 질적 저하 가능성.

## 2.1. 서비스명

## 2.2. 핵심 가치 (Core Value)

재학생의 **커리큘럼**(학습 과정)과 **프로젝트**(학습 성과) 유기적 연결(Weave)을 통한 성공적 **커리어**(미래 설계) 구축 지원 학과 특화 커뮤니티 플랫폼.

본 서비스의 핵심 가치는 세 가지 차원으로 구체화.

1. **투명성 (Clarity):** 트랙 로드맵을 통한 복잡한 커리큘럼 이어짐(Weave) 시각화 및 모든 학생 대상 명확한 학습 경로 제공
2. **증명성 (Evidence):** 내 프로필 내 이수 과목과 프로젝트 갤러리 성과물 연계(Weave)를 통한 개인 역량 객관적 증명 포트폴리오 구축
3. **협력성 (Collaboration):** 팀 빌딩을 통한 필요 역량 보유 학생 연결(Weave) 및 성공적 프로젝트 협업 지원

# 3. UX 설계 및 기능 정의

## 3.1. 페이지 상세 구조

### 1. 홈 (Home)

- **역할:** 서비스 랜딩 페이지 및 주요 콘텐츠 요약 대시보드.
- **사용자 경험:** 로그인 시 최초 접근 페이지로서 서비스 전반적 활성도(Activity) 체감 제공. 사용자의 핵심 기능 유도 역할 수행.
- **기능:** `프로젝트 갤러리` 내 인기(조회수, '좋아요' 기반) 프로젝트 및 `팀 빌딩` 최신/마감 임박 모집 공고 노출.

### 2. 미디어 커리큘럼 (Category Page)

**역할:** 학과 커리큘럼의 '탐색', '계획', '참조'를 위한 핵심 기능 허브.

- **[세부 페이지 2.1] 트랙 로드맵 (Track Roadmap)**
    - **UI:** 5개 트랙 선택 탭(필터) 및 인터랙티브 그래프 영역.
    - **사용자 경험:** 탭 선택을 통한 5개 트랙 과목 지도 즉각적 전환 및 비교 가능. 그래프 확대/축소/패닝 지원.
    - **기능:**
        - JavaScript 기반 그래프 라이브러리(예: React Flow) 활용 과목 간 선수과목 관계 스킬 트리 형태 인터랙티브 그래프 시각화.
        - 그래프 내 개별 과목 노드(Node) 클릭 시 해당 과목 핵심 정보(과목 개요, 선수과목) 제공 모달(Modal) 팝업.
- **[세부 페이지 2.2] 나만의 커리큘럼 짜기**
    - **UI:** 4년 8학기(1-1 ~ 4-2) 그리드(Grid) 및 드래그 가능(Draggable) 과목 목록.
        
        (이수 완료(Locked), 수강 중(Highlight), 수강 예정(Plannable) 상태 시각적 구분)
        
    - **기능:**
        - **성적표 CSV 업로드**: 사용자 기이수 내역 CSV 파일 파싱(Parsing)을 통한 그리드 내 이수 완료 상태 과목 자동 구성. 서비스 온보딩(Onboarding) 시 사용자 초기 설정 부담 획기적 감소.
        - 과목 목록에서 수강 예정 과목의 그리드 드래그 앤 드롭(Drag & Drop) 배치.
        - 완성 플랜의 `Curriculums` (MongoDB) 컬렉션 저장.
        - 저장 시 `Prerequisites` DB(Postgres) 조회를 통한 선수과목 이수 규칙 위배 여부 실시간 유효성 검사 수행.
- **[세부 페이지 2.3] 공유된 커리큘럼 확인**
    - **UI:** 카드형 갤러리 레이아웃. (예: "GI 트랙 4학년 선배의 플랜")
    - **기능:** 나만의 커리큘럼 내 `isPublic: true` (공개) 설정 타 사용자 커리큘럼 플랜 목록 조회.

### 3. 커뮤니티 (Category Page)

- **역할:** 학생 간 학술 활동(프로젝트) 공유 및 상호작용(팀빌딩) 허브.
- **[세부 페이지 3.1] 프로젝트 갤러리 (Project Gallery)**
    - **UI:** 카드형 갤러리 (Behance, GitHub 스타일), 상세 필터 사이드바.
        
        (필터 다중 선택(Multi-select) 지원)
        
    - **사용자 경험:** 다중 필터(예: '3학년' + 'VC 트랙' + 'Python' 태그)를 통한 원하는 프로젝트 정밀 탐색 가능.
    - **기능:** 프로젝트 데이터 CRUD (Create, Read, Update, Delete) 제공.
- **[세부 페이지 3.2] 팀 빌딩**
    - **UI:** 게시판(Forum) 레이아웃, 상세 필터 사이드바.
    - **기능:** 팀원 모집 게시물 CRUD 제공. 모집 중, 모집 완료 상태(Status) 변경 기능.

### 4. 내 프로필 (My Profile)

- **역할:** 개인화 대시보드 및 포트폴리오 페이지.
- **사용자 경험:** 외부(예: 취업 지원) 공유를 위한 사용자 '출구(Exit Point)' 역할 수행.
- **UI:** 좌측(프로필 정보 - 이름, 트랙, 한줄소개, GitHub/LinkedIn 링크), 우측(탭 기반 콘텐츠 영역).
- **기능:**
    - **'프로필 Markdown(.md) 업로드'**: `.md` 형식 마크다운 파일 업로드를 통한 '소개' 및 '주요 프로젝트' 섹션 자동 구성.
    - `프로젝트 갤러리` 등록 본인 계정 프로젝트 목록 조회 (탭 1: Projects)
    - `팀 빌딩` 등록 본인 계정 게시물 목록 조회 (탭 2: Posts)
    - `나만의 커리큘럼 짜기` 저장 플랜 목록 조회 및 '공개' 여부 설정(Toggle) 기능 (탭 3: Curriculums)
    

# 4.  개발 계획 및 기술 스택

## 4-1. 개발 계획

3단계(P1: MVP, P2: Enhancement, P3: Future) 구분을 통한 점진적 구현.

### M1. MVP: 4-Week Target

**Must Have (필수 기능) - 서비스 핵심 가치 제공**

1. **사용자 인증 및 관리**
   - 회원가입 / 로그인 / 로그아웃
   - 학번, 이름, 트랙 정보 입력
   - 비밀번호 암호화 (Bcrypt)

2. **트랙 로드맵 (Read-Only)**
   - `Tracks`, `Courses`, `Prerequisites` 테이블 데이터 사전 입력 (Pre-seeding)
   - JS 그래프 라이브러리(React Flow) 활용 인터랙티브 그래프 조회
   - 5개 트랙 선택 탭 필터
   - 과목 노드 클릭 시 기본 정보(과목명, 개요, 선수과목) 모달 표시
